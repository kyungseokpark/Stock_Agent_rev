이 문서는 **WO-001~005 구현 이후 추가되는 모든 수정/개선 요청을 한곳에 누적**한다. 새 파일을 만들지 않고 여기에 항목(수정-N)으로 계속 추가한다.

## 수정-1. 한글 폰트 깨짐 (최우선)
- 증상: 좌측 "분야(섹터)" 라벨·섹터 체크 항목이 `??(??)` 등으로 깨짐.
- 원인: 동적 생성·순수 `tk` 위젯(Listbox/Label/Checkbutton)에 한글 폰트 미적용.
- 지시:
  1. 시작 시 전역 기본폰트 Malgun Gothic 고정:
     ```python
     import tkinter.font as tkfont
     for n in ("TkDefaultFont","TkTextFont","TkMenuFont","TkHeadingFont","TkTooltipFont"):
         try: tkfont.nametofont(n).configure(family="Malgun Gothic", size=11)
         except Exception: pass
     root.option_add("*Font", ("Malgun Gothic", 11))
     ```
  2. 모든 `tk.*` 위젯에 `font=("Malgun Gothic",11)` 명시.
  3. ttkbootstrap 테마 적용 **이후** 폰트 재설정. 소스 UTF-8 저장.
- 검증: 좌측 네비 전 항목 한글 정상(깨짐 0).

## 수정-2. 대비 — 회색/아이보리 위 흰 글씨 절대 금지
- 증상: ① 회색 배경+흰 글씨(헤더/비선택 탭), ② **툴팁이 아이보리(연노랑) 배경에 흰 글씨라 안 보임**, ③ 콤보/스핀/입력창 밝은 배경에 밝은 글씨.
- **불변 규칙: 글자색은 실제 배경에 맞춘다. 밝은 면엔 어두운 글씨, 어두운 면엔 밝은 글씨. 밝은 면 위 흰 글씨 금지.**
- 지시:
  - 툴팁: 배경 `#2A3036` + 글자 `#F2F4F6`(아이보리 기본 제거), 테두리 `#3D454E`.
  - 콤보 드롭다운·스핀박스·입력창: 다크 통일(배경 `#2A3036`/글자 `#F2F4F6`). 밝은 배경 강제 위젯은 글자 `#1F2428`.
  - 테이블 헤더: 배경 `#2A3036` + 글자 `#F2F4F6` 굵게, 경계선 `#3D454E`.
  - 탭: 선택=배경 `#2A3036`+글자 `#FB923C`(오렌지)+하단 오렌지바, 비선택=글자 `#C8CFD6`↑.
  - 본문 대비 ≥ 7:1. 비활성 위젯도 글자 대비 확보.
- 검증: 툴팁·드롭다운·입력창·헤더·탭 모두 또렷.

## 수정-5. 스크리닝 진행률 표시
- 증상: 실행 중 몇/몇·단계·현재 종목이 안 보임(스피너만). 원인: 엔진이 진행을 UI로 안 보냄.
- 지시:
  1. `run_screen(config, top_n=None, progress_cb=None)` — 루프에서 `progress_cb({"market","stage","current","total","ticker"})` 호출. 단계: 데이터 수집 → 신호 계산 → 세력/종합 → 후보 선정. 단계 전환 시 total 갱신.
  2. 워커가 콜백 안에서 `message_queue.put(("progress",payload))`, `_poll_queue`가 진행바/상태라벨 갱신(메인스레드). **throttle**: 5종목/100ms마다 1회.
  3. 진행바 `mode="determinate", maximum=total, value=current` + % 텍스트.
  4. 상태 텍스트 예: `[국장] 데이터 수집 320 / 2,740 (12%) · 삼성전자 · 경과 0:45`. 둘 다 실행이면 `[1/2 국장]`/`[2/2 미국장]`. 완료/오류 시 마무리 표기.
  5. 위치: 창 하단 **전체폭 상태바**(항상 보임). 사이드바 길면 스크롤.
- 검증: 진행바 0→100%, 카운트·단계·종목 실시간. 창 크기 변해도 안 가림. 대량에도 안 버벅임.

## 수정-6. "점수 기준" 탭 — 변수명 금지, 사람 말 + 실제 값
- 증상: `vol_asym`, `top_bear_vol`, `{'cmf':0.2,...}` 등 변수/딕셔너리 그대로 노출 → 비전문가 이해 불가.
- **불변 규칙: 화면에 변수키·중괄호 dict·영문키 노출 금지. config 값은 내부에서 읽되, 화면엔 "한글 이름 + 쉬운 설명 + 실제 값(%/점)"으로 변환 출력.**
- 표시 내용(이 문장 템플릿에 숫자만 끼움):

  **차트 점수**(0~100, "지금 사기 좋은 기술적 자리인가"): 추세(평균선 위)·거래량·모멘텀·MACD·가격위치·손익비·감점 합산. 최종 = 차트기본 70% + 수축패턴 15% + 상대강도 10% + 주봉추세 3% + 시장분위기 2%. 판정: 80↑ 강력 후보 / 65↑ 후보 / 50↑ 관찰 / 그 아래 제외.

  **패턴 보너스**: 돌파 +5, 눌림목 반등 +3, 20일선 회복 +2, 신고가 근처 +2, 과매도 반등 −3, 떨어지는 칼날 −10, 패턴 불명 −2. 샘플(가짜) 데이터 제외.

  **세력 유입(실수급 데이터 있을 때)**: 외국인 순매수 강도 24%(거래대금 15%↑ 순매수면 만점)·기관 순매수 강도 18%·외국인 연속매수일 13%(7일이면 만점)·외국인 지분율 증가 13%(20일 +2%p 만점)·기관 연속매수일 8%·순매수 가속 8%·개인 안 쏠림 8%·신용 안 늘고 상승 8%.

  **세력 유입(데이터 없을 때, 가격·거래량 추정)**: 오를 때 거래량 > 내릴 때 거래량 18%(가장 큼)·자금흐름 20%·거래량 누적방향 16%·매집봉 비율 12%·내릴 때 거래량 마름 10%·거래량 급증 10%·박스 돌파+지지 8%·횡보 매집 6%.

  **위치 보정**: 바닥권 ×1.2 · 보통 ×1.0 · 단기 급등 직후 ×0.8 · 고점권 ×0.6.

  **분산(빠져나가는) 감점**: 고점 대량 큰 음봉 −15 · 급등 후 거래량 안 줄고 하락 −10 · 고점 윗꼬리 반복 −8 · 돌파 다음날 반납 −7 · 대량인데 종가<시가 −5.

  **흐름(단계) 보너스**: 바닥횡보→거래량증가→하락시감소→돌파→지지→눌림→재상승, 단계당 +2.5, 최대 +15.

  **종합 점수**: 차트 60% + 세력 40%. (세력 못 구하면 차트만)

- 종목 클릭 분해표도 한글 라벨: 추세/거래량/모멘텀/MACD/가격위치/손익비/감점/기본점수/수축패턴/상대강도/차트점수/세력유입%/위치보정/분산감점/흐름보너스/종합점수.
- 탭 하단 "직접 확인 권장"(자동 점수 외, WO-003 §8): 호가창·체결강도, 종토방/텔레그램 언급, 내부자 매도 공시.
- 검증: 탭에 영문 변수·dict 없음. 모든 항목 "한글 이름+설명+실제 값". config 바꾸면 숫자만 바뀜. 60대 비전문가 이해 가능.

## 수정-7. 국장 실수급(외국인·기관) 자동 확보 — 3소스 시도 + 되는 곳 기억
- 증상: **국장**인데 "실수급 데이터 없음"으로 떠서 프록시로 빠짐. (미국장은 원래 실수급이 없으므로 손대지 않음 — 프록시 정상.)
- 적용 범위: **국장(코스피/코스닥) 전용.** 미국장은 그대로 둠.
- 지시:
  1. **소스 체인(순서대로 시도, 한 곳이라도 성공하면 그 데이터 사용):**
     - ① **pykrx** (`stock.get_market_trading_value_by_date` 등 외국인/기관 순매수)
     - ② **네이버 금융** 종목별 외국인·기관 순매매(frgn 페이지 크롤 — 기존 네이버 크롤 코드 재사용)
     - ③ **KRX data.krx.co.kr** 직접 호출(pykrx 원천 엔드포인트)
  2. **되는 곳 기억:** 성공한 소스를 상태파일(`data/cache/flow/_source.json`)에 저장 → **다음 실행부터 그 소스를 먼저** 시도. 그 소스가 실패하면 다시 체인 전체 재탐색하고 성공 소스로 갱신.
  3. **마지막 거래일 기준:** 오늘이 주말/공휴일/장중이라 당일 확정치가 없으면 **가장 최근 거래일**의 확정 수급을 사용. → **휴장은 "없음" 사유가 아니다**(사유에서 제외).
  4. **requirements**: `pykrx`(+ 크롤용 `requests`,`beautifulsoup4` 이미 사용) 추가·설치 확인. EXE 빌드에도 포함.
  5. **티커 형식 정규화:** KRX/네이버는 `005930`(6자리), 내부는 `005930.KS/.KQ` → 변환 일치.
  6. **실패 사유 표기(국장만):** 3소스 모두 실패한 진짜 사유만 표시 — "pykrx 미설치 / 네트워크 차단 / 응답 파싱 실패". **휴장·주말은 사유에 넣지 않음**(마지막 거래일로 처리되므로).
- 검증:
  - [ ] 국장 실행 시 외국인·기관 순매수 실값이 채워지고 `세력유입 source=real`로 표기.
  - [ ] pykrx 막아도 네이버/KRX 중 하나로 성공하면 real 유지.
  - [ ] 성공 소스가 `_source.json`에 기록되고 다음 실행에서 우선 시도.
  - [ ] 주말/공휴일에 실행해도 마지막 거래일 수급으로 정상(“없음” 아님).
  - [ ] 3소스 전부 실패할 때만 사유 표기 후 프록시 폴백.
