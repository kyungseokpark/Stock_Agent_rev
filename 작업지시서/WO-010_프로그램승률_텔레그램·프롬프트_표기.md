# WO-010: 프로그램 누적 승률 계산 + 텔레그램·Claude 프롬프트 말미 표기 (2026-07-04)

## 1. 목적
매 실행마다 **백데이터(추천 이력 + 사후 추적 결과)로 프로그램의 누적 승률을 계산**하고,
그 결과를 **텔레그램 요약 메시지와 Claude 검수용 프롬프트의 마지막**에 항상 표기한다.
사용자가 "이 프로그램의 추천이 지금까지 얼마나 맞았는지"를 매 알림에서 즉시 확인할 수 있게 한다.

## 2. As-Is (현재 상태)
- `src/performance_tracker.py`에 추천 이력 DB(`data/screener_history.db`) 저장·추적 로직 존재:
  - `save_recommendations()` — 매 실행 시 Top N 추천을 DB에 저장 (이미 동작).
  - `update_tracking_results()` — 최대 10거래일 사후 가격으로 목표가/손절 터치를 갱신
    (`performance.auto_update_tracking: true`로 KR/US 모두 활성).
  - `summarize_tracking_performance()` — 평균 5/10일 수익률, 1차 목표 터치율, 손절 터치율,
    동시터치 비율 산출. **단, "승률" 단일 지표는 없음.**
  - `build_performance_summary_text()` — 위 요약을 텍스트로 변환.
- 텔레그램 요약 메시지: `stats["performance_summary_text"]`가 있으면 말미에 붙음(이미 연결됨).
  **단, 승률이 아닌 터치율 나열이라 직관성이 떨어짐.**
- Claude 검수용 프롬프트(`build_claude_mobile_prompt`): **성과/승률 정보가 전혀 없음.**
- 세력주 Top 5(세력 점수 기준, WO 이후 추가된 기능)는 추적 대상에 저장되지 않음
  (종합 Top N만 `save_recommendations`로 저장).

## 3. To-Be (요구 사항)
1. **승률 지표 정의 및 계산** (`summarize_tracking_performance` 확장):
   - `win_rate`(주 승률): 추적 완료(10거래일 경과) 추천 중
     **1차 목표가를 손절보다 먼저 터치**(`first_hit in {target1, target2}`)한 비율(%).
   - `positive_10d_rate`(보조): 10일 시점 수익률 > 0 비율(%).
   - `win_rate_by_market`: KR/US 분리 승률.
   - 표본 수(`completed_count`)를 항상 병기 — 표본 30건 미만이면 "참고용(표본 부족)" 문구 추가.
2. **텔레그램 요약 메시지 말미 표기** (`build_performance_summary_text` 확장):
   ```
   [프로그램 누적 승률]
   승률(1차 목표 선도달): 58.3% (완료 24건)   ← 표본 30건 미만 시 "참고용(표본 부족)" 병기
   10일 수익률 플러스 비율: 62.5%
   평균 5일/10일 수익률: +1.2% / +2.8%
   손절 선터치율: 25.0%
   ```
   - 완료 표본이 0건이면 "아직 승률 표본이 없습니다(추적 중 N건)"로 대체. 절대 섹션 자체를 생략하지 않음.
3. **Claude 검수용 프롬프트 말미 표기** (`build_claude_mobile_prompt` 확장):
   - 프롬프트 마지막(검수 요청 항목 뒤)에 동일한 승률 블록을 추가하고,
     검수 요청 항목에 **"4. 위 프로그램 누적 승률을 참고해 이번 추천의 신뢰 수준을 평가해 주세요."** 추가.
   - `config["_run_stats"]["performance_summary_text"]` 재사용 (별도 재계산 금지 — 실행당 1회 계산).
4. **매 실행 보장**: GitHub Actions/로컬 실행 모두에서 스크리닝 완료 → 추적 갱신 →
   승률 계산 → 메시지/프롬프트 표기 순서가 항상 수행되는지 확인
   (`run_screen` 내 `write_performance_outputs` 호출이 텔레그램 전송보다 선행함을 유지).
5. **(선택 확장) 세력주 Top 5 추적**: `save_recommendations`에 `list_type` 컬럼("composite"/"force")을
   추가해 세력주 랭킹도 저장하고, 승률을 종합/세력 각각 분리 산출. 스키마 마이그레이션 필요
   (기존 DB 호환: 컬럼 없으면 "composite"로 간주).

- [ ] KR/US 각각 실행 시 해당 시장 승률이 표기됨 (전체 누적과 시장별 병기).
