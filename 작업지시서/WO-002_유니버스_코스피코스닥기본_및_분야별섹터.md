# WO-002 · 유니버스 & 스크리닝 — 코스피+코스닥 기본 + 분야별(섹터)

목적: ① 국장 실행 시 **기본값으로 KOSPI+KOSDAQ 전체** 스크리닝, ② **분야(섹터)별 선택 스크리닝**(의약·바이오/헬스케어/반도체 등).
선행 의존: WO-001 P0-1(가격 캐시·배치), P0-2(신호 단일 계산), P0-3(섹터 무효). 두 기능 모두 같은 KR 데이터 소스(pykrx/네이버)를 쓰므로 **함께 구현**.

---

## A. 코스피 + 코스닥 기본화

### As-Is
- `config_kr.yaml universe.mode: "kospi"`(코스피 단독 기본). `kospi_kosdaq` 모드는 이미 구현됨.
- 코스피 ~1,015 / 코스닥 ~1,725 = 합 ~2,740종목.

### To-Be / 구현
1. `config_kr.yaml` → `universe.mode: "kospi_kosdaq"` 로 변경.
2. 데스크톱은 config mode를 역매핑해 라벨을 잡으므로 **config만 바꾸면 "KOSPI + KOSDAQ"가 기본 선택**(추가 코드 불필요). `UNIVERSE_OPTIONS["kr"]`에 매핑 존재 확인(있음).
3. 성능: WO-001 P0-1 캐시·배치 필수. 진행률 `Processing 540/2740` 표기, 실패/유동성통과/후보 수 요약탭 노출.
4. 유동성 임계값 캘리브레이션(WO-001 P1-3): 코스피/코스닥 `avg_dollar_volume_20d` 분포 측정 → 권장 시작값 코스피 50억, 코스닥 20억. 단일/분리 구조 택1.
5. full_result 보존 상위 N(100~200) 고려. 콤보로 KOSPI/KOSDAQ 단독·관심종목 전환 유지.

### 검증
- [ ] 앱 기동 시 "분석 대상" 기본 = "KOSPI + KOSDAQ", `unique_tickers ≈ 2,700+`.
- [ ] 캐시 미적중 ≤20분 / 적중 ≤2분. 콤보 전환 정상.

## B. 분야별(섹터) 스크리닝

### B-1. 섹터 데이터 파이프라인 (핵심)
- 소스 우선순위: ① **pykrx**(KRX 업종) → ② **네이버 금융 업종**(`sise_group.naver`, 기존 크롤 패턴 재사용) → ③ yfinance `info["sector"]`(보조).
- 산출물: `data/sector_map_kr.csv` (`ticker,sector,industry,sector_source,updated_at`). **WO-001 공통 캐시 정책** 적용(주1회 갱신).
- 신규 `src/sector_loader.py`: `build_kr_sector_map(force_refresh)`, `attach_sectors(universe_df)`.

### B-2. 표준 섹터 분류 (한글 라벨로 정규화)
원천 업종명이 제각각 → `data/sector_taxonomy.yaml`(키워드→표준섹터)로 통일. 미매핑은 "기타" + 로그.

| 표준 섹터 | 포함 키워드 예 |
|-----------|----------------|
| 의약·바이오 | 제약, 바이오, 의약품, 생명공학 |
| 헬스케어·의료기기 | 의료기기, 진단, 헬스케어 |
| 반도체 | 반도체, 장비, 소부장 |
| 2차전지·소재 | 2차전지, 양극재, 전지소재 |
| IT·소프트웨어 | 소프트웨어, 인터넷, 게임, SI |
| 자동차·부품 | 완성차, 자동차부품 |
| 금융 | 은행, 증권, 보험, 지주 |
| 화학·에너지 | 화학, 정유, 에너지, 신재생 |
| 조선·기계·방산 | 조선, 기계, 항공우주, 방산 |
| 소비재·유통 | 식음료, 화장품, 유통, 의류 |
| 기타 | 미해당 |

### B-3. 설정 (`config_kr.yaml`)
```yaml
sector_filter:
  enabled: false      # 기본 전체, UI에서 켬
  include: []         # 예: ["의약·바이오","헬스케어·의료기기"]
  refresh_days: 7
```

### B-4. 적용 위치
- 유니버스 로드 직후 `universe = attach_sectors(universe)` → `sector_filter.enabled & include`면 `universe[universe.sector.isin(include)]`. 필터 후 종목 수 stats 반영.

### B-5. UI (desktop_app.py)
- 사이드바 "분석 대상" 아래 **"분야(섹터)" 멀티선택**(체크/리스트박스) + "전체". 선택 시 `config["sector_filter"]={enabled:True, include:[...]}`.
- 결과 테이블/상세에 `sector` 컬럼 표기.

### B-6. 섹터 분산 정상화
- 섹터 채워지면 `portfolio.max_per_sector` 자동 동작. "기타"/Unknown 캡 적용 여부 `portfolio.cap_unknown_sector`로 선택.

### 검증
- [ ] `sector_map_kr.csv` 생성, 코스피+코스닥 섹터 채움률 ≥90%(미매핑=기타).
- [ ] "의약·바이오,헬스케어" 선택 → full_result 모든 행 sector ∈ 선택값.
- [ ] 미선택 결과 = 전체 스캔 동일. 섹터 분산 시 섹터별 후보 ≤ max_per_sector.
