## P0 — 치명

### P0-1. 공통 데이터 레이어: 가격(OHLCV) 일일 캐시 + 배치 다운로드
- 위치: `main.py run_screen()` 루프, `src/data_loader.py fetch_ohlcv()`
- 현황: 종목마다 `yf.download(timeout=600)` **순차** 호출, 캐시 없음. KOSPI+KOSDAQ ~2,700종목이면 수십 분~수 시간 + 레이트리밋 위험.
- 수정지시:
  1. **일일 캐시** `data/cache/ohlcv/{ticker}.parquet`(메타 `last_fetched`). 당일 재다운로드 금지.
  2. **배치 다운로드** `yf.download(list, group_by="ticker", threads=True)` 50~100개 묶음, 실패분만 개별 재시도.
  3. 개별 timeout 600→15~30초, 전체 상한(예: 20분) + 진행률 로깅.
  4. 실패/스킵 종목 수 `stats` 집계 → 요약탭/로그 표기.
- **이 캐시 레이어는 WO-002(섹터), WO-003(세력 수급데이터)에서 동일 패턴으로 재사용**한다(`data/cache/` 하위로 통일).
- 검증: 코스피+코스닥 1회, 캐시 미적중 ≤20분 / 적중 ≤2분.

### P0-2. `generate_signal` 이중 호출 제거
- 위치: `main.py` 1차 루프 + 재랭킹에서 전 종목 재호출 → 연산 2배.
- 수정지시: 히스토리 수집 → RS rank 산출 → **종목당 단 1회** `generate_signal(rs_rank 포함)`. (또는 재랭킹 시 scoring만 rs 반영해 재산출.)
- 검증: 결과 동치, 호출 수 종목당 1회.

### P0-3. 섹터 데이터 전무 → 섹터 기능 무효
- 위치: `data/universe_*.csv`(sector 공란), `src/portfolio.py`(빈 섹터=`Unknown`으로 캡 제외).
- 영향: "섹터 분산" 무효, 분야별 스크리닝 불가. → **WO-002에서 섹터 파이프라인으로 해결**(여기선 의존성만 명시).

## P1 — 정확성·일관성

### P1-1. `min_score_for_candidate` / UI "최소 점수" 슬라이더 미적용 (데드 컨트롤)
- 위치: `src/recommendation_quality.py select_top_candidates()` — 점수 하한 게이트 없음(정렬+샘플/손익비 필터만).
- 수정지시: `min_score = config["screening"]["min_score_for_candidate"]` 하한 필터 추가, 후보가 top_n 미만이면 경고 로깅.
- 검증: 슬라이더 80 → 80 미만 종목 후보에서 제외.

### P1-2. 이벤트(실적) 필터 항상 무효
- 위치: `signal_engine.py` → `earnings_context(..., context.get("earnings_dates"))`인데 `earnings_dates` 주입된 적 없음 → 절대 차단 안 함.
- 수정지시: (a) 실적일 소스(yfinance `Ticker.calendar` 등) 연결·주입, 또는 (b) 미지원이면 설정 비활성화하고 UI/요약에 "미동작" 명시.

### P1-3. 유동성 필터 변수명·단위 혼동 + 코스닥 임계값 재튜닝
- 위치: `config_kr.yaml filters.min_avg_dollar_volume_20d: 5e9`(=KR 원화 거래대금인데 이름은 dollar), `universe_loader.liquidity_metrics()`.
- 수정지시: 주석 명확화(KR=원화 거래대금). 코스피/코스닥 임계값 분리 가능 구조(`filters.kr.kospi`, `filters.kr.kosdaq`). 기본값은 WO-002에서 분포 측정 후 결정.

### P1-4. 샘플 데이터 혼입 표기
- 위치: `data_loader.build_sample_ohlcv`. custom 모드 한정이나 full_result에 남아 혼동.
- 수정지시: 운영모드(kospi/kosdaq/combined)에선 샘플 생성 차단 명문화, full_result에서 `data_source=sample` 구분 표기.
