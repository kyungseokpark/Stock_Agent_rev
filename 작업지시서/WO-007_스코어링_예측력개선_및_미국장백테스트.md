## 1. 코드 수정 (스코어링 예측력)

패치 초안은 `분석리포트/*.patch` 에 있다. Codex는 이를 참조하되 repo 최신 상태에 맞게 반영·테스트한다.

### 1-1. `src/regime.py` — 급락장 조기 감지
As-Is: `last > ma200 and ma50 > ma50_prev` → risk_on / `last < ma200 and ma50 < ma50_prev` → risk_off / 그 외 neutral.
To-Be: 단기 급락(10일 −5% 또는 20일 고점대비 −7% 또는 종가 < MA20)을 `short_break` 로 정의하고,
- risk_on 조건에 `and not short_break` 추가,
- risk_off 조건에 `or (short_break and last < ma50)` 추가.

```python
ma20 = float(close.tail(20).mean())
ret10 = float(close.iloc[-1] / close.iloc[-11] - 1) if len(close) > 11 else 0.0
drawdown = last / float(close.tail(20).max()) - 1
short_break = (last < ma20) or (ret10 <= -0.05) or (drawdown <= -0.07)
```
효과: risk_off 전환 시 `signal_engine.py`의 기존 게이트가 `valid=False`로 매수콜 자체를 차단.

### 1-2. `src/scoring.py` — 레짐 승수 + 상대강도 하드게이트
context 블록 최종점수 산출 직후에 추가:
```python
# (A) 상대강도 하드게이트: 지수 대비 리더십 없으면 Candidate 진입 불가
min_rs = _num((config or {}).get("ranking", {}).get("min_rs_rank", 0))
if min_rs and rs_rank < min_rs:
    final = min(final, 64)
# (B) 레짐 승수: 기존 2% 가중은 무력 → 점수 자체를 축소
regime_state = str(context.get("regime_state", "neutral"))
regime_mult = {"risk_on": 1.0, "neutral": 0.90, "risk_off": 0.70}.get(regime_state, 0.90)
final = int(round(final * regime_mult)); final = max(0, min(100, final))
```

### 1-3. `src/scoring.py` — 떨어지는 칼날 방지 + RR 왜곡 차단
```python
risk_reward_raw = _num(targets.get("risk_reward"))
risk_reward = min(risk_reward_raw, 5.0)          # 비정상 RR 점수 인플레 차단
...
penalty -= int(bool(ma20) and bool(ma50) and ma20 < ma50) * 8   # 하락정배열 붕괴 감점
penalty -= int(risk_reward_raw > 10) * 8                        # 비현실적 목표/손절 기하
```

## 2. 미국장 백테스트 러너 (신규 `run_us_backtest.py`) — Codex 구현

> **미국장 전용.** 위 코드 수정의 예측력 개선을 계량 검증한다. `src/backtest.py:run_picks_backtest` 를 그대로 호출하는 얇은 CLI 래퍼로 만든다(백테스트 로직 신규 작성 금지, 재사용).

### 2-1. 기능 요구사항
- **인자**: `--config`(기본 `configs/config_us.yaml`), `--label`, `--lookback`(기본 `2y`), `--top-n`(기본 3), `--benchmark`(기본 `SPY`), `--max-tickers`(0=제한없음, 스모크용), `--offline`(캐시 parquet 직접·신선도 무시), `--out`(기본 `output/backtest`), `--compare`(baseline summary.json 경로).
- **시세 로드**: 기본은 `read_ohlcv_cache`→실패 시 `fetch_ohlcv`. `--offline` 이면 캐시 parquet 직접 로드(과거 검증·네트워크 불필요). `data/cache/ohlcv/` 재사용.
- **유니버스**: `src/universe_loader.load_universe(config)` 사용. `--max-tickers` 로 상한.
- **벤치마크**: SPY(또는 ^GSPC) 로드해 `run_picks_backtest(..., benchmark_df=...)` 로 전달(RS/레짐/벤치마크 대비 수익 계산).
- **호출**: `run_picks_backtest(tickers, config, market="us", top_n_per_day, lookback_period, data_by_ticker=histories, benchmark_df, progress_cb)`.
- **출력**: 콘솔 요약 + 저장 `{label}_trades.csv`, `{label}_equity.csv`, `{label}_summary.json`.
- **비교(`--compare`)**: baseline summary.json 을 읽어 지표별 `baseline → 현재 (▲/▼ 차이)` 표로 출력.
- **지표**: `_summarize` 산출값 전부 — 거래수, 승률, 기대R, 손익비(PF), MDD, 샤프, 총수익%, 벤치마크%, IS/OOS 거래수, OOS 기대R.
- CSV 인코딩 `utf-8-sig`, 콘솔 메시지 한글.

### 2-2. 실행 절차 (패치 전/후 A/B)
```bat
:: 1) 패치 적용 전(baseline) — git stash 등으로 원본 상태에서
python run_us_backtest.py --label baseline

:: 2) WO-007 §1 패치 적용 후(patched) — baseline 대비 비교
python run_us_backtest.py --label patched --compare output/backtest/baseline_summary.json

:: (네트워크 없이 캐시로 과거검증)  python run_us_backtest.py --label patched --offline
:: (빠른 스모크)                    python run_us_backtest.py --label smoke --max-tickers 40 --offline
```

### 2-3. 판정 기준 (Acceptance) — 미국장, 패치 후가 패치 전 대비
- **OOS 기대 R 유지 또는 개선** (핵심; 하락 시 회귀·재검토).
- **승률·손익비(PF) 비열화** (동등 또는 개선).
- **MDD 악화 없음** (레짐 승수·게이트가 하락구간 노출을 줄여야 함).
- 총수익%가 벤치마크(SPY)% 대비 열위로 전환되지 않을 것.
- 스모크(40종목/offline)로 먼저 파이프라인 정상 동작 확인 후 전체 유니버스 실행.

## 3. 국장 특수상황 처리 방침 (중요)

**국장은 이번 국면을 기술 스코어로 강제 교정하지 않는다.** 하이닉스·삼성처럼 **펀더멘털이 살아있는데 수급(외국인 매도·원화 약세)으로 눌리는** 특수상황은 순수 기술적 점수로 예측이 어렵다. 따라서 국장은:
- §1 패치(레짐 조기감지·RS 게이트·칼날 방지)로 **"하락장 매수 자체를 줄이는"** 방어에 집중(신규 매수콜 억제).
- **점수 모델을 국장 상승 예측용으로 오버피팅하지 말 것.** §1-4 가중치 조정 이상으로 국장 전용 튜닝을 추가하지 않는다.
- 백테스트/검증은 **이번 WO 에서는 미국장만** 수행. 국장 백테스트·펀더멘털(수급/실적) 결합 스코어는 **별도 WO 로 분리**(후속).
- (후속 검토 아이디어, 이번 범위 아님) 세력유입(`force_inflow`)·펀더멘털 데이터를 국장 점수에 결합, 국장 전용 레짐 벤치마크(KOSPI/KOSDAQ) 정합성 점검.
