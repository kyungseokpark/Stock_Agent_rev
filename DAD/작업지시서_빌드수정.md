### 🔴 문제 1 — ttkbootstrap 완전 누락 (최우선)

**증상**: 실행 즉시 아무 반응 없이 종료  
**원인**: `ttkbootstrap`이 exe 내부 PYZ, `_internal` 폴더, `base_library.zip` 어디에도 포함되지 않음. `hiddenimports`에만 등록하면 PyInstaller가 테마 데이터 파일(.json, 이미지 등)을 수집하지 못함. `collect_all()`을 사용해야 함.

**수정 위치**: `DAD/StockAgentDAD.spec`

**수정 내용**:

```python
# spec 파일 상단에 추가
from PyInstaller.utils.hooks import collect_all

ttkb_datas, ttkb_binaries, ttkb_hidden = collect_all('ttkbootstrap')

# Analysis 블록 수정
a = Analysis(
    ...
    datas=[
        (str(project_root / "configs"), "configs"),
        (str(project_root / "data"), "data"),
        (str(project_root / "tickers.csv"), "."),
        (str(project_root / "tickers_kr.csv"), "."),
        (str(project_root / "매뉴얼.html"), "."),
        *ttkb_datas,          # ← 추가
    ],
    binaries=[
        *ttkb_binaries,       # ← 추가
    ],
    hiddenimports=[
        *ttkb_hidden,         # ← 추가
        "bs4",
        "html5lib",
        "lxml",
        "matplotlib",
        "matplotlib.backends.backend_tkagg",
        "pyarrow",
        "pyarrow.compute",
        "pyarrow.lib",
        "pyarrow.parquet",
        "pykrx",
        "yfinance",
        # "ttkbootstrap" ← 이 줄은 collect_all로 대체됐으므로 제거 가능
    ],
    ...
)
```

### 🔴 문제 2 — UPX 압축으로 인한 백신 오탐 차단

**증상**: 일부 PC에서 실행 즉시 Windows Defender 또는 백신이 격리/차단  
**원인**: 현재 spec에서 `upx=True`로 설정되어 있고 exe에 UPX 시그니처가 실제로 적용돼 있음. PyInstaller + UPX 조합은 백신의 휴리스틱 탐지에 걸리는 대표적인 케이스.

**수정 위치**: `DAD/StockAgentDAD.spec` — EXE 블록과 COLLECT 블록

```python
exe = EXE(
    ...
    upx=False,    # ← True에서 False로 변경
    ...
)
coll = COLLECT(
    ...
    upx=False,    # ← True에서 False로 변경
    upx_exclude=[],
    ...
)
```

### 🟠 문제 3 — 런타임 쓰기 경로가 _internal 내부에 고정됨

**증상**: `C:\Program Files` 등 쓰기 권한이 제한된 경로에 설치하거나, 읽기 전용 드라이브에서 실행 시 로그/캐시/결과 저장 실패로 오류 발생  
**원인**: `logs/`, `output/`, `data/cache/` 폴더가 `_internal` 하위에 번들로 포함돼 있어, 런타임에 해당 경로에 파일을 쓰려 할 때 권한 오류 발생 가능. PyInstaller 배포 앱에서 쓰기가 필요한 경로는 exe 옆 또는 사용자 AppData로 분리해야 함.

**수정 위치**: `desktop_app.py` 및 경로 관련 유틸 파일 (예: `src/market_config.py` 등)

**수정 내용**: 경로 결정 로직을 아래 패턴으로 통일

```python
import sys, os

def get_base_dir() -> str:
    """실행 파일 기준 루트 경로 반환 (PyInstaller/일반 실행 모두 대응)"""
    if getattr(sys, 'frozen', False):
        # PyInstaller 실행 환경: exe가 있는 폴더
        return os.path.dirname(sys.executable)
    else:
        # 개발 환경: 소스 루트
        return os.path.dirname(os.path.abspath(__file__))

def get_resource_dir() -> str:
    """읽기 전용 리소스 경로 (configs, data 원본 등) — _internal 내부"""
    if getattr(sys, 'frozen', False):
        return sys._MEIPASS
    return os.path.dirname(os.path.abspath(__file__))

BASE_DIR    = get_base_dir()      # 쓰기 가능 경로 (exe 옆)
RESOURCE_DIR = get_resource_dir() # 읽기 전용 번들 리소스

# 사용 예시
LOG_DIR    = os.path.join(BASE_DIR, "logs")
OUTPUT_DIR = os.path.join(BASE_DIR, "output")
CACHE_DIR  = os.path.join(BASE_DIR, "data", "cache")
```

**확인 필요 사항**: 현재 코드에서 `logs/`, `output/`, `data/cache/`를 참조하는 모든 경로 문자열을 위 패턴으로 교체.

### 🟡 문제 4 — 실행 실패 시 에러가 완전히 묵살됨

**증상**: `console=False` 빌드이므로, 어떤 예외가 발생해도 사용자에게 아무 메시지가 표시되지 않음. 디버깅 불가.  
**원인**: PyInstaller `console=False` + 예외 처리 없음.

**수정 위치**: `desktop_app.py` — 메인 진입점

**수정 내용**: 최상위 try/except로 예외를 로그 파일에 기록하고 간단한 에러 창 표시

```python
import sys, os, traceback, tkinter as tk
from tkinter import messagebox

def main():
    # ... 기존 앱 실행 코드 ...
    pass

if __name__ == "__main__":
    try:
        main()
    except Exception as e:
        # 에러를 파일에 기록
        log_path = os.path.join(os.path.dirname(sys.executable) if getattr(sys, 'frozen', False) else ".", "error.log")
        with open(log_path, "w", encoding="utf-8") as f:
            f.write(traceback.format_exc())
        # 사용자에게 에러 창 표시
        try:
            root = tk.Tk()
            root.withdraw()
            messagebox.showerror(
                "실행 오류",
                f"프로그램 시작 중 오류가 발생했습니다.\n\n{e}\n\n상세 내용: {log_path}"
            )
        except Exception:
            pass
        sys.exit(1)
```
